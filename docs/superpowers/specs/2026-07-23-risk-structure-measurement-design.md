# s030 账本风险结构测量——beta 暴露分解 + 选币相关性（预注册）

日期：2026-07-23 · 状态：预注册（判据先于数据）· 类型：测量研究（零上线改动）

## 1. 背景与目的

用户令「全面检查系统 + 分散投资风险」。体检结论：当前唯一未量化的大风险向量是
**系统性 crypto beta**（普跌时 12 格同吃止损，防线只有 shock brake 只关不开 + 软止损）与
**选币板块聚堆**（代码无任何跨币相关性约束）。两个候选方向——(a) beta 对冲腿、
(b) 选币相关性/板块上限——机制上地位不同：(a) 叠在账本之上不动选币不伤 alpha；
(b) 与 grid-fitness 排名直接对冲，且历史「分散广度」战役已判死一次（瓶颈=每窗少数好币
非 lane 数，choose_n 战役 2026-07-23 进行中）。

用户裁定路线：**先量化现状再定目标**。本研究测两个数：账本回撤/波动里系统性成分占多少
（裁 a），选币相关性高不高、网格机制是否已把它中和（裁 b）。一次测量，两线同裁。

## 3. 方法

### Layer B（主）——账本级 beta 分解

1. **账本收益序列**：每格 `pnl_ratio` 等权（实盘 cap 均等），乘以 `cap_equity_frac=0.2451`
   得权益口径。主口径记入**退出桶 = 含 rt+12h 的 12h 桶**（UTC 00/12 对齐）；
   稳健性检验：记入 rt 桶、以及 24h 日桶（吸收 ±12h 退出时刻近似误差）。
   注：逐格无退出时刻，12H 格退出 ∈ [rt, rt+12h]，此为已知近似，判据以主口径为准。
2. **回归**：`ledger_ret_t = α + β·mkt_ret_t + ε`，分窗 OLS；mkt 三口径各跑一遍：
   票池中位数（mkt_r15 取非重叠 :00/:15/:30/:45 采样复利到桶）、BTC 1h 收盘复利、ETH 同。
   报每窗 β、R²。
3. **崩盘窗 MDD 分解**：HOLD-A 账本累计曲线（算术累加，pp 惯例）取峰谷区间
   [t_peak, t_trough]；系统性成分 = β_HOLD-A × 同区间市场累计收益；特质 = MDD − 系统性；
   报系统性占比（clamp [0,1]）。
4. **尾部截面**（比线性 β 更贴威胁模型）：12h 桶按市场收益分档
   （<−3% / −3~−1 / −1~+1 / +1~3 / >+3%），报每档格数、均值 pnl、退出原因分布
   （固定止损/pv 主动/资金费率/窗口结束/回撤止盈）——直接验证「普跌时止损聚集」。

### Layer A——选币相关性

每 rt 的并发组合 = rt′∈(rt−12h, rt] 开的格（≤12 个，含同币 cap2 重复）。
组合内两两 1h 对数收益相关（窗 [rt−7d, rt)，≥120 根有效才计），得每 rt 平均 pairwise ρ；
另算组合对 BTC 平均 ρ。报每窗 ρ̄ 分布（中位数/p90）。不引外部板块标签
（相关性本身当板块代理，免标签噪声）。

## 4. 预注册判据（先写死，后看数）

| 结果 | 裁定 |
|---|---|
| 全窗 R²<0.2 **且** HOLD-A MDD 系统性占比<1/3 | **beta 对冲判死**（保险费>保额，关线） |
| 任一窗 R²≥0.4 **或** HOLD-A 系统性占比≥50% | **beta 对冲立项**（进对冲腿设计实验，才谈对冲工具/比例/成本） |
| 中间带 | 只测尾部条件版（shock brake 触发时段条件 β）再裁 |
| Layer A ρ̄中位≥0.5 **且** beta 对冲判死 | **相关性上限判死**（网格已中和相关性；与 choose_n 广度判死互证） |
| Layer A ρ̄中位≥0.5 **且** beta 对冲立项 | 两线取一：**先对冲**（不动选币不伤 alpha），相关性上限缓议 |
| Layer A ρ̄中位<0.5 | 相关性上限无标的，关线 |

R² 主判口径 = 票池中位数因子；BTC/ETH 口径供对冲工具选型，不参与生死判。
