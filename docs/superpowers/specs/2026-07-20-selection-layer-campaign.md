# 选币层战役 v1(2026-07-20)

**目标**:最差窗(OOS 逆风)Calmar 实质改善,其余窗不输——参数层已定案无解(重扫v2 终审),
这是选币层的第一次系统性进攻。**基线=旧现值全参数 + 现行排名(等权 [1,1,1])**,诚实引擎。

## 设计输入(全部来自已归档实证,见 memory binance-param-resweep)

- **sbuck 四格地图**:OOS 全部亏损(−3.70pp vs 全窗 −3.27pp)由事件币段(s>1.3)贡献;
  「非事件币 × 大盘回调」格四窗唯一全正。s=meanTR₅/medianTR₅(波动形状,栈外维度)。
- **头名溢价**:3/4 窗递补劣于 rank-1 → 只许"改排名本身",禁止排后否决(无替换税结构)。
- **现行栈语义**:Reg_v2_5(偏离回归线深度)+ Sgcz_5(距近期高点回撤)= 找回调;Er_2(震荡均衡度)。
  三者等权 rank-sum,HL 时代定的,币安诚实引擎上从未复核。
- **死假设(不复测)**:R(中位动量)门控×s(三窗三死法:按住赢家/替换税/双重);无条件 s 过滤
  (事件币四窗净 +1.9pp,是被补偿的风险溢价);funding 零门(旧引擎中性,丢的格在赚钱);
  方向性 pv;tier2_cap=1;所有几何/估计量变体。

- 条件分支:若 drop-one 显示某因子负贡献 → 追加对应 double-one 臂;否则不跑。

## 判定规则(ex ante)

- 主序=最差窗 Calmar;采纳门槛=**全窗 Calmar 不输锚(±0.3 噪声带)且 OOS 实质改善**
  (Calmar ≥+0.5 或 ret ≥+0.5pp)。
- 单窗爆胜+他窗崩 = regime 噪声,直接归档(第 N 次)。
- 存活臂 → HOLD-A(2025-02~03 腰斩)/HOLD-B(2024-10~11 牛市)终审,双窗不输才改 config。
- 胜臂须补 sbuck 式逐格归因(赢的钱来自哪格)+ pick 换血率审计。

## 执行

- 通道:全 canonical——`SW.BT_STRATEGY/BT_FACTORS` 参数化传入(select cache 键含
  strategy_config,臂间天然不串);S_shape 经 cal_factor 在 worker 内原生计算,无 monkeypatch。
- 资源:冷选币 workers=3(缓存 MISS 规则)、仿真 workers=4;pv_cache 每窗臂间共享。
- 顺序:窗主序 OOS→W1→W2→IS,每窗 5 臂,断点续跑;锚校验失败响亮中止。
- 测速纪律:OOS 首臂计时后据实报总时长再续。

**W1 处决轮**:无Er −1.33/−6.25/−1.2(锚 +2.35/−2.04/7.3)——OOS 冒头第五次被顺风窗击毙,
W2/IS 裁撤(必要条件已死,纯尸检不值 3h)。noReg/noSgcz 余窗同理裁撤(OOS 恶化=资格死)。

**终判:现行等权三因子 [1,1,1] 是选币排名层的甜点,战役零改动收官。**
- **各因子分工实测**:Reg=OOS 防线(拆掉 2 破网+固损×3);Sgcz=拥挤过滤(拆掉费率止损 85 笔冲前三);
  Er=**regime 稳定器**(拆掉 OOS 多赚 2.1pp 但 W1 崩 3.7pp+MDD×3——纯 dip 排名=接飞刀策略)。
- **+S_shape 死因=Goodhart**:sbuck 说"自然选中且 s≤1.3 的格全窗正",但按 s 最小化排名捞进来的是
  匀速阴跌币/死水币(pv 掉出退出前三,窗口结束 573);等权第 4 因子换血 76%,远非"边际倾斜"。
  描述性分段 ≠ 可优化目标;s 保留仪表身份。
- **方法论沉淀**:换血% 是"改良 vs 换策略"的量尺(>40% 就不是同一策略的改良);OOS 单窗冒头
  已 5 连毙(n=4/median_k/rv_k/门/无Er),此模式今后直接按海妖处理。
- 运维注:战役中 macOS mediaanalysisd 突发负载触发 load 熔断(22.4),hard_kill 漏杀 3 孤儿 worker
  (PPID=1,各 100% CPU)——孤儿清扫循环需加固(候补:按 PPID=1 且 cmdline 含 spawn_main 直杀)。
