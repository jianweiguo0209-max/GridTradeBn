# 选币层战役 v1(2026-07-20)

**目标**:最差窗(OOS 逆风)Calmar 实质改善,其余窗不输——参数层已定案无解(重扫v2 终审),
这是选币层的第一次系统性进攻。**基线=旧现值全参数 + 现行排名(等权 [1,1,1])**,诚实引擎。

## 设计输入(全部来自已归档实证,见 memory binance-param-resweep)

- **sbuck 四格地图**:OOS 全部亏损(−3.70pp vs 全窗 −3.27pp)由事件币段(s>1.3)贡献;
  「非事件币 × 大盘回调」格四窗唯一全正。s=meanTR₅/medianTR₅(波动形状,栈外维度)。
- **头名溢价**:3/4 窗递补劣于 rank-1 → 只许"改排名本身",禁止排后否决(无替换税结构)。
- **现行栈语义**:Reg_v2_5(偏离回归线深度)+ Sgcz_5(距近期高点回撤)= 找回调;Er_2(震荡均衡度)。
  三者等权 rank-sum,HL 时代定的,币安诚实引擎上从未复核。
- **死假设(不复测)**:R(中位动量)门控×s(三窗三死法:按住赢家/替换税/双重);无条件 s 过滤
  (事件币四窗净 +1.9pp,是被补偿的风险溢价);funding 零门(旧引擎中性,丢的格在赚钱);
  方向性 pv;tier2_cap=1;所有几何/估计量变体。

## 判定规则(ex ante)

- 主序=最差窗 Calmar;采纳门槛=**全窗 Calmar 不输锚(±0.3 噪声带)且 OOS 实质改善**
  (Calmar ≥+0.5 或 ret ≥+0.5pp)。
- 单窗爆胜+他窗崩 = regime 噪声,直接归档(第 N 次)。
- 存活臂 → HOLD-A(2025-02~03 腰斩)/HOLD-B(2024-10~11 牛市)终审,双窗不输才改 config。
- 胜臂须补 sbuck 式逐格归因(赢的钱来自哪格)+ pick 换血率审计。
