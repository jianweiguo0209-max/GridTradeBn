# 反事实选币评价器（cf_eval）设计规格

**日期**：2026-07-23 · **状态**：设计已口头批准，待用户审阅本文
**性质**：⚠**回顾性测量工具**——量已发生的路径，禁作预测（边界见 §6，违者结论无效）

---

## 1. 背景与重定性

用户目标：**横向量化比较几个币 12h 内已实现走势的网格适配度，从而评估选币优劣**。
不预测任何东西，只准确评价已发生的事实。**量的是走势的固有适配度（路径属性，
不含我方退出链的税）——主分数=E0 纯网格口径（2026-07-23 用户定）**。

上次"失败"的账要拆开算：

- **死的是预测线**（Phase2：可预测⊥可获利、截面平坦）——与本目标无关。
- **本目标真正的前科** = 2026-07-22 评价分工具效度验证：横向排序被证伪
  （5 组合×4 窗×7000 格，标签排序 vs 引擎 pnl 排序 Spearman +0.10、W1 完美反转 −1.00），
  工具被降级为"证伪+纵向监控"。三个死因：
  1. **尺子不准**：calib_score 两因子线性代理 R²=0.21，丢 79% 方差，最致命是止损链
     时序交互（好形状路径早触 pv → 引擎实收 0，标签分照高）。
  2. **被测差异本来就小**：合理选币组合间 1~5bp/格 vs 单格噪声底 134bp，n=350 连
     引擎 pnl 自己都排不出序（bootstrap ≤1.1σ）。
  3. **设计浪费了事后优势**：各组合独立抽样，窗口运气不配对，市场 β 噪声全额进入。

本设计对症：**死因1** 用真引擎反事实模拟当分数（R²=1 按定义）；**死因3** 用同窗全池
配对（市场运气逐窗对消，逐窗结论零推断噪声）；**死因2** 治不了，诚实面对（§6.4）。

## 2. 核心评价器

对一个选币轮 `(rt, offset)`：取该轮 PIT 票池（与选币同口径，24h qv 前 55% 过滤），
对**每个池内币**在已实现 `[rt, rt+12h)` K 线上跑两遍标定引擎：

| 口径 | 配置 | 含义 |
|---|---|---|
| **pnl_E0** | 现役几何（band3×ATR/cmin16），纯网格无退出链，跑满 12h | **走势固有网格适配度 = 主分数**（路径属性，不含退出链税） |
| **pnl_s030** | 同几何 + 全止损链（pv−0.01/tr2%/stop0.03）+ funding | 系统实收，副刻度（对照"固有的肉系统吃到多少"） |
| 差值 | pnl_E0 − pnl_s030 | 该币该窗的止损链税 |

**引擎复用 `data/score_research_2026-07-21/s030_calib.py` / ablation 引擎**（已对 ec_*
全 2,796 格逐位自检），不新写引擎——choose_n 教训：harness 外重写必翻车；此处引擎
已存在且已锚验，只把驱动从"抽样格"铺到"全池"。入场价沿用标定引擎的 middle_5≈开盘
近似（评价用途可接受，Stage E 备案）。

燃料/毒药/汇率（现有 calib_score 组件，`holdout_gate.py` stage L/F 管线）保留为
**诊断列**，解释"为什么差"（死水/漂移/被止损链收走），**不参与排序**。

## 3. 指标定义

每轮 `(rt, offset)` 出数，按窗/周聚合。K = 该轮实选数（随 config choose_symbols，勿钉死）；
"真实 top-K" = 该轮全池按 **pnl_E0** 排序前 K。排序类指标一律用主口径 pnl_E0；
pnl_s030 列并读，回答"固有的肉系统吃到多少"。

| 指标 | 定义 | 回答 |
|---|---|---|
| 选币 alpha | mean pnl_E0(选中) − mean(全池) | 选得比池平均好多少 bp（市场运气逐窗对消） |
| 捕获率 | \|选中 ∩ 真实 top-K\| / K，按窗聚合 | 每窗少数好币抓中几个（已证瓶颈=对症指标） |
| 遗憾 | mean(真实 top-K) − mean(选中) | 丢在桌上的 bp |
| 止损链税 | (pnl_E0 − pnl_s030)，选中 vs 池 | 持续监控 E0–E4 消融结论是否在漂 |
| 池体温 | 全池 pnl_E0 分布（中位/p25/p75/top-K 均值） | "没赚钱是没好币，还是没选中好币" |
| 诊断 | 选中 vs 池的 z(cross1)/z(drift)/汇率 | 差在哪个轴（衔接现有评价分工具读法） |

**A/B 选币器比较**（研究驱动）：同池分歧格上的配对差（共同选中精确抵消），
报告须带分歧格数 n 与 CI。

## 4. 两阶段交付

### Phase 1：研究驱动
- 驱动器遍历六窗（W1/W2/OOS/IS/HOLD-A/B）的选币轮，数据用回测缓存 K 线；
  PIT 票池过滤可预计算。
- **验收锚（双口径）**：同 2,796 标准格重算——pnl_E0 对 `ablation/geo_*` 的
  **m30_c16 格点**（3×ATR×16 格、E0 口径，即 s030 几何；band3 下 flex=12<16 恒被
  cmin16 兜底，固定 16 格与 config 语义等价。⚠不是 `ab_*` 的 E0 列——那是旧几何
  2×ATR/10 格）、pnl_s030 对 s030_calib parquet，均 **diff==0**（锚不平先查保真度，
  不得替换）。
- **首个产出物**：现役 rank_sum 六窗成绩单（选币 alpha/捕获率/遗憾/止损税/池体温
  逐窗一行）。
- 算力预算：~150 币×24 轮/日×60 日 ≈ 21.6 万格/窗（估算）；超预算按 offset 降采样
  （配对设计下统计无损）。多进程守 `__main__` + workers≤4。

### Phase 2：实盘巡检
- 每日批处理：对过去 24h 已收盘的轮，join `selection_snapshots`（实际选中记录；
  若快照缺票池字段，按选币同规则 PIT 重建）。
- K 线拉取：**每币每日一次**（轮间数据重叠，按日取按轮切），~150–280 次调用/日，
  走 paced fetcher（250ms），权重可忽略；避开选币轮高峰时段。
- **验收锚**：选中币反事实 pnl 对实盘账本——pv 止损格复现精度参照 recon 先例
  （5/5 复现、Δ≤0.15pp）；总体 Δ 中位 ≤0.5pp 且残差可解释为执行/费差。
  账本锚只覆盖 pnl_s030 口径（E0 无实盘对应物，由 P1 研究锚背书；两口径共享
  引擎核与数据通路）。
- 干跑一日通过后再谈常驻（cron/报表落库后议，本 spec 不含）。

## 6. 预注册边界（违者结论无效）

1. **逐窗数字=事实；"选币器 A 长期更好"=多窗聚合宣称**；任何由此改策略的前瞻动作
   照走三道门+留出（docs/研究方法论.md）。
2. **禁止把反事实排名回喂选币**——事后分喂前瞻 = 已判死的预测线，此门焊死。
3. 反事实假设显式化：未交易币按引擎规则成交（candle-cross 假设，$3k 规模无市场
   冲击）；实盘 qv 为代理（(o+c)/2·vol，差 ~0.16%）只影响 pv 边缘触发，量化后接受。
4. 预期管理：合理选币器间差 1~5bp/格（s030 口径实测），工具会经常报"≈池平均"
   ——**这是信息不是故障**；主价值 = 遗憾分解 + 红旗（票池塌陷/选币退化立刻显形），
   不是给两个都还行的选币器分高下。E0 口径量纲更大（止损链吃掉 71% 期望 pnl：
   E0 均值 3.5bp/格 vs E4 1.0），固有差异分辨率天然更高，但预期同上、不预设更乐观。
