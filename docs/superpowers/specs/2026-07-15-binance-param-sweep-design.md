# 币安参数重扫设计（2026-07-15）

> 动机：现行策略参数（固定止损 0.045、trailing 0.15/0.015、布网带宽 2 / 间距上限 0.04、
> pv thr 0.005×mult 3、gearing 3.4）**全部是在 Hyperliquid 上调出来的**，随迁移原样搬到币安。
> 但币安与 HL 在三个决定性维度上不同：**费率更贵**（VIP0 maker 2bps/taker 5bps vs HL base
> 1.5/4.5bps）、**票池宽 5 倍**（400+ 币动态池 vs $1M 地板后 ~75 币）、**最小名义额/杠杆上限
> 按币分档**。已知这些会改变最优点（例：GP 的"窄带高换手胜出"结论建立在更便宜的费率上；
> 换手的边际收益在币安被手续费侵蚀更多）。本 spec 定义一次系统性重扫。

## 0. 前置修复：回测 pv 语义 ≠ 实盘 pv 语义（阻塞项）

**现状（bug）**：`backtest_run._simulate_grid_task` 调 `simulate_grid_engine` 时不传
`pv_spike_df` → 引擎回落到「只用 12h 持仓窗内的 48 根 15m bar、`rolling(100, min_periods=1)`」
算量能基线（= 窗口内扩张均值，开窗头几根样本数仅 1-2，且完全看不到开仓前的量能水位）。
**实盘**（`execution/signals.py`，spec 2026-07-07-pv-legacy-semantics-live）取**原生 15m
K 线 n+8 = 108 根 ≈ 27h 前置历史**，`rolling(100)` 为真·25h 滚动基线。

**后果**：pv 是最大单项退出驱动（币安四窗实测砍掉 ~49% 的格），回测在一个**实盘不存在的
信号**上评估它——pv 归因不可信，且在此信号上扫 pv 阈值选出的值搬到实盘即错。GP 的扫参
harness（`data/tiercmp/pv_n_sweep._flags_n`）绕过了这个坑（拼 27h 前置历史再算），但两个
项目的 `backtest_run` 都没接线——口径分裂由此产生。

**修复（本 spec 的第 0 步，先于扫参）**：`assemble_grid_tasks` 按 27h 前置历史逐格预算
`pv_spike_df`（源自已载入的完整 1m 序列），随 data_task 下传，`_simulate_grid_task` 转交引擎。
引擎侧无需改动（`pv_spike_df` 入参早已存在，优先级高于内部近似）。

**影响面（如实记录）**：修复后所有回测数字（含 2026-07-15 跑出的币安四窗基线
+8.94%/+14.56%/+7.08%/+4.03%/+14.14%）**作废重出**。GridTradeGP 仓库有同一 bug；两边此前的
对比表在「同为近似 pv」下内部一致，方向性结论（币安票池优势）不受影响，但如需刷新为诚实
口径，须同步移植该修复并重跑 GP 四窗（本 spec 不含，另议）。

## 1. 范围

**扫**（本次）：
- 仿真侧：固定止损、trailing(k×floor)、pv(thr×mult)、资金费率止损、gearing
- 布网几何：atr_range_multiplier × grid_count_min（2D）、grid_spacing_max

**不扫**（本次显式排除，保口径可比）：
- 选币侧：因子/权重、票池 top%（0.55）、换仓周期（12H）、choose_symbols
- 风控闸：tier2_cap（=2）、shock（0.025/4h/2h）、黑名单
- 账户级：ACCOUNT_LEVERAGE（=5.0，cap 定档另议；gearing 扫的是单格名义部署倍数，
  两者经 `derive_frac` 耦合——gearing 若改，AL 定档须重算 frac）

## 2. 评估口径

- **主排序：Calmar**（年化 ÷ |组合 MDD|，用户定 2026-07-15）
- **并列键：四窗 Calmar 最小值**（worst-window）——防「一窗暴赚三窗平庸」的解胜出
- **否决线**：任一窗出现破网/爆仓 → 淘汰；最差窗 MDD 劣于现值 1.5 倍 → 标红人工复核
- 同时报：每窗收益、四窗均值、触网胜率、平均成交数（费率敏感度）、退出归因迁移、
  是否**支配现值**（四窗全 ≥）
- 收益口径 = `summarize()` 的 `portfolio_return`（12 offset lane 各自复利、等权平均），
  与全部历史基线可比；MDD/年化/Calmar 在 lane 结构上按平仓时刻构造组合净值曲线计算
  （已实现口径，格内浮亏不可见——如实注记）

## 4. 参数网格（Pass 1，单旋钮围绕现值；粗体=现值）

| 族 | 网格 | 臂数 | 怀疑依据 |
|---|---|---|---|
| 固定止损 `stop_loss` | 0.030 / 0.035 / 0.040 / **0.045** / 0.055 / 0.065 / 0.080 / OFF | 8 | IS 窗仅 36 格却吃掉 −134pp、单均 −4.95%——线位与实际损失脱节；小币跳空在币安更狠 |
| Trailing `k × floor` | k[0.10, **0.15**, 0.25] × floor[0.010, **0.015**, 0.025] + OFF | 10 | 币安触网更频繁（触网胜率高 8-13pp），floor（唯一敏感自由度）最优点大概率漂移 |
| PV `pnl_thr × mult` | thr[0, 0.0025, **0.005**, 0.0075, 0.010] × mult[2, **3**, 4] + OFF | 16 | 最大单项成本；且必须在**修复后的实盘同源信号**上扫（见 §0） |
| 资金费止损 | 0.0005 / 0.0010 / **0.0015** / 0.0025 / OFF | 5 | HL 每小时结算 vs 币安 8h 结算——阈值照搬最可疑；且币安侧它是正收益（31 格 +7.8pp），HL 侧 0 格触发 |
| 布网几何 2D | band[1.5, **2**, 3, 4, 5] × count_min[5, **10**, 20, 30] | 20 | GP 的「窄带高换手」结论建立在 1.5bps 费率；币安 2/5bps 侵蚀换手收益，最优点可能回退宽带 |
| 间距上限 `grid_spacing_max` | 0.02 / 0.03 / **0.04** / 0.06 / 0.08 | 5 | 直接决定每格成交频次 × 费率 |
| Gearing | 2.4 / 2.9 / **3.4** / 3.9 / 4.4 | 5 | 线性放大 pnl 振幅与爆仓风险；币安票池更杂、波动更大 |

**Pass 1 ≈ 69 臂 × 4 窗。**

**Pass 2（坐标下降）**：取 Pass 1 各族 Calmar 最优（满足否决线）组成新基线，在新基线上
**重扫敏感度最高的 3 族**（预计 pv / 几何 2D / 固定止损，以 Pass 1 的 Calmar 极差排序决定）
≈ 44 臂 × 4 窗。捕获一阶交互（例：间距变窄 → 成交变多 → 费率吃掉利润 → pv 门槛最优点跟着漂）。
两轮收敛后定候选 → 留出窗验收。

## 5. 架构

**新增 `gridtrade/backtest/sweep.py`**（纯离线、入库）
| 函数 | 职责 |
|---|---|
| `preload_window(cache, universe, ws, we, *, workers)` | 每窗**一次**：选币（磁盘缓存命中）→ shock 剔轮 → cap2 递补分配 → 载选中币 1m 全序列 → 逐格切持仓 bars/funding 切片 + **27h 前置历史 pv 源**。返回 `WindowData`（几何无关的 raw 格集） |
| `build_arms(family, baseline)` | 参数网格（纯数据，无副作用）→ `[Arm(label, kind, overrides)]` |
| `run_arm(wd, arm, *, workers)` | 按 arm 重算几何（几何族才需）+ 重算 pv 尖峰（pv 族才需，否则复用预计算）+ 仿真 → 明细 DataFrame |
| `metrics(df, days)` | Calmar / 年化 / 组合 MDD / 收益 / 触网胜率 / 破网爆仓数 / 平均成交 / 退出归因 |

**性能设计**：pv 尖峰按 `(mult, n, period)` 键预计算并跨臂复用——非 pv 族的 53 臂共享同一份，
省掉最贵的内循环（15m 重采样 + rolling）。几何族只重算 `calc_grid_params_v2`（廉价纯函数），
bars/funding/pv 全部复用。

**新增 `scripts/sweep_run.py`**（CLI）
```
.venv/bin/python -m scripts.sweep_run --family stop|trail|pv|funding|geom|spacing|gearing|all \
    --windows W1,W2,OOS,IS --out data/sweep/
```
输出 `data/sweep/{family}_results.csv`（arm × window × 全部指标），`data/` gitignored，
**脚本入库**——修掉 GP 时代「调参脚本躺在 gitignored `data/tiercmp/`、参数如何调出无据可查」的硬伤。

**新增 `tests/backtest/test_sweep.py`**
- pv 修复的回归钉：前置历史存在时，窗口内近似会误报的尖峰不再误报（合成序列：窗口前高量、
  窗口内中量 → 近似口径把开窗首根判为尖峰，实盘口径不判）
- `metrics` 的确定性与 lane 数学（对拍今日四窗脚本的算法）
- 基线臂 = 现值时 `build_arms` 产出恒等于 config 现值（防口径漂移）

## 6. 防口径漂移守卫（GP 传统，强制）

- `sweep.py` 顶部**断言钉死当前 config 值**（`stop_loss==0.045`、`trailing==0.15/0.015`、
  `band==2`、`spacing_max==0.04`、`gearing==3.4`、`pv==0.005/3/100/15min`）——config 一改
  脚本立刻炸，杜绝「基线悄悄变了、报告仍按老基线解读」
- **单旋钮原则**：每臂只改一族，其余全取实盘现值
- 选币口径锁死实盘现值（top-55% + 2026-07-15 新选币语义（选币层交易额过滤已删）+ cap2 +
  shock 0.025/4h/2h）
