# 量化研究方法论 · GridTradeBi

> 从 2026-07 的多轮参数/信号研究中固化。适用范围:本项目任何"改动能不能提升策略"的研究——
> 选币、止损、择时、几何、仓位。**这是协议,不是建议**;跳过任何一环,结论不可信。

---

## 0. 一句话

**带机制的假设,用最狠的证伪去打,留出定生死,机制串成地图。**

目标不是找赢家,是诚实测绘"哪里有边、哪里没有"。一个干净关掉一个方向的负结果就是产品。

---

## 1. 核心循环

```
假设(必带机制) → 最便宜的能证伪它的测试 → 存活才升级 → 留出定生死 → 机制解释结果 → 更新地图
```

- **假设必带机制**:不做纯拟合。先问"如果它真,什么物理让它真"。没有机制的信号不测。
- **证伪优先,不是证实**:主动找"它怎么可能是错的",不是找"它对的证据"。
- **机制解释结果并预测下一个**:每个结果给机制解释,好的机制会预判下一个实验(如"V型崩会弹→
  切进去反噬"预判了熔断/冷却/time_decay 三个死法)。

---

## 2. 三道门(廉价→昂贵,存活才升级)

| 门 | 成本 | 作用 | 判据 |
|---|---|---|---|
| 标签门 | 秒 | 信号对路径标签的 IC / 桶提升 | 标签门都输 → 直接埋,省后续算力 |
| 引擎门 | 分钟 | 抽样格喂真引擎,信号↔pnl | 标签赢≠引擎赢(Phase2血训);噪声底~134bp/格 |
| 组合战役 | 小时 | 真选币+12offset+Calmar | 单旋钮臂,锚逐位复现 |
| 留出 | 决定性 | HOLD-A/B 零接触裁决 | **唯一的法官(见§3)** |

**标签门都赢不了的东西,绝不为它烧一小时组合回测。**

---

## 3. 判定窗 + 留出双验(承重墙)

- **判定窗(W1/W2/OOS/IS)**:只用来探索和调参。
- **留出(HOLD-A 2025腰斩 / HOLD-B 2024牛市)**:只验收、不参与任何选择。**裁决只在这里算数。**
- **铁律**:任何在判定窗调过参/挑过的候选,不过留出就不算赢。
- **历史战绩(2026-07)**:留出斩了 7 个"判定窗漂亮"的候选(pvloose/idioZ3/cmin20/p12_cross1/
  shock-flow/offset结构/time_decay),多个直接**反号**。判定窗里的边默认是 regime 噪声。

---

## 4. 让结论可信的纪律

1. **预注册判据**:pass/fail 门槛写在跑之前(脚本 docstring 里)。赛后不许挪门柱、不许"看到
   offset 3/11 再说它们特殊"(赛后画靶=多重比较噪声)。
2. **锚逐位复现**:每个回测先重跑基线并 byte 级比对存档——防管线悄悄坏、防配置漂移、保证同口径。
3. **反事实对照**:不是"X赢没赢",是"对**正确基线**赢没赢"。动作 EV 对"让现役止损链处理"而非
   裸持有;恢复对照组;Oracle 天花板。
4. **量纲 + 统计功效**:噪声底、bootstrap 显著性、多重比较(N候选里~N/8个会四窗巧合同号)、
   有效N。知道差异何时只是噪声。绝对量纲阈值撞"币间波动率不齐"(判别器两代死法)。
5. **配对消融**:同样本只改一变量(E0-E4/单旋钮臂),把因果从混杂剥出。
6. **诚实归因、认错**:失败当失败报带证据;自己的假设/算错(如组合口径 ÷lane数非÷格数)当场公开纠正。
7. **廉价可逆断点**:窗级并行、parquet 断点、Monitor 等完成、结果清楚就 kill。多进程守 __main__
   (macOS spawn 陷阱)+ workers≤4。

---

## 5. 默认先验与何时停

- **非凡主张默认是噪声,直到留出证否。** N 次留出斩首后,下一个候选先验极低——诚实说出来,
  即使还在跑。
- **区分"数据说停" vs "用户要继续"**:给诚实的收口推荐,但把决定权交用户。
- **均值回归策略的元教训**:所有"降风险/找预测"的直觉(割快、回撤暂停、止损冷却、订单流择时)
  在均值回归里**是反的**——回撤/下插/止损后正是边所在(抄底/抓反弹)。**边在结构里,不在预测里。**
  活下来的只有顺策略本性的结构常量(s030几何/止损;分散广度),不是任何预测信号。
