> 给新接手者的一页纸。详细操作见 [回测使用文档.md](回测使用文档.md)(必读三章:顶部**保真度里程碑**、**§5.3 当前基线**、**§5.4 解读陷阱**)。

## 0. 一句话定位

这套回测**与实盘共用同一批生产函数**(选币/建网/引擎/退出判定单源),已用 mainnet 5 个真实网格做三层对账:**建网 byte 精确、成交逐笔对齐、退出归因一致**。可以信它——但只信 **2026-07-19 之后**的数字(此前六项保真度 bug 使历史回测系统性虚高,已全部修复并作废旧数,详见保真度里程碑章)。

## 1. 环境准备(新克隆四步)

```bash
python3.9 -m venv .venv && .venv/bin/pip install -r requirements.txt   # ①
.venv/bin/python -m scripts.fetch_lot_sizes        # ② stepSize 缓存(data/ 不入库,必跑一次)
cp .env.example .env                               # ③ 填 BINANCE_API_KEY/SECRET(杠杆过滤用,只读权限即可)
.venv/bin/python -m pytest -q                      # ④ 全绿(~1000 条)才算环境 OK
```

- **没有 API key** → `BT_MIN_LEVERAGE=0` 可跑,但票池与实盘断代(多 ~170 个低杠杆币),数字不可与默认口径混比。
- 所有命令带 `TZ=Asia/Shanghai`(选币时区口径)。

## 3. 窗口与评判约定(别自创)

```python
WD = {'W1': ('2025-08-15','2025-10-14',61),  'W2': ('2025-10-15','2025-12-14',61),
      'OOS':('2026-01-01','2026-02-28',59),  'IS': ('2026-03-01','2026-06-30',122),
      'HOLD-A':('2025-02-01','2025-03-31',59),'HOLD-B':('2024-10-01','2024-11-30',61)}
```

- 调参只用前四窗;**HOLD-A/B 是留出窗、只做终审**,谁在上面调参谁负责。
- 主序 = **最差窗 Calmar**;爆仓>0 一票否决;破网人审。

## 4. 五条铁律(违反必翻车)

1. **回测对不上实盘 → 先查引擎保真度,再怀疑环境,最后才调参**(三次全套扫参数字作废均因顺序反了:pv 语义→首触丢弃→pv 前视)。
2. **每次扫描必带锚臂**(参数=现值的臂),它必须逐位复现 §5.3 基线;不复现 = 管线有偏、全表作废。
3. **负收益窗别看 Calmar**(符号扭曲,看 ret+MDD);**band 族 Calmar 虚高**(宽带=低建仓率,须 ret+MDD 联判);cmin 是高噪声维(flex_count=4×band 恒等式使 cmin 恒绑定)。
4. **本机资源**:scratchpad 脚本必须带 `if __name__=='__main__'` 守卫(macOS spawn 会内核 panic);**cache miss 的选币(尤其 IS 大窗)workers≤3**,暖缓存才可 4-8;自备 load 看门狗(>18 连续两次即杀)。
5. **已验证否定的假设别重跑**(spec/memory 有档):24H 周期、方向性 pv、tier2_cap=1、shock thr 0.015、band 加宽、pv thr −0.015、候选A 三改动。

## 5. 当前参数与基线(对账基准)

**最新基线 s030 ★（2026-07-24 冻结）**:band 3 / cmin 16 / stop 0.03 /
trailing 0.3/0.02 / pv −0.01×3×100 / funding 0.0015 / gearing 3.4 /
shock 0.025/4h/2h / period 12H / tier2_cap 2。后续默认回测均与此版比较。

**s030 ★ 六窗基线**（每格为 ret% / Calmar / MDD%）：W1 −2.83 / −3.7 / 4.29；
W2 +6.31 / 17.4 / 2.55；OOS +1.85 / 4.7 / 2.59；IS +13.11 / 11.2 / 3.97；
HOLD-A −2.36 / −2.9 / 4.70；HOLD-B +1.58 / 4.1 / 2.39。前四判定窗已用当前代码
重放对齐；HOLD-A/B 使用终审冻结结果，不为普通调参重复打开。

**手工报告入口（2026-07-25）**：
`TZ=Asia/Shanghai BT_WORKERS=3 PYTHONWARNINGS=ignore .venv/bin/python -u -m scripts.six_window_backtest <名称>`。
顺序固定 W1爆雷→W2震荡→OOS逆风→IS磨涨→HOLD-A腰斩→HOLD-B牛市；逐窗闸门，失败即停止，
报告落 `data/result/<因子组合>/<名称>/report.html`。
默认 `--mode traditional` 失败即停；需完整六窗对照时用 `--mode full`，任一窗失败则最终状态仍为 FAILED。

## 6. 实盘对账现状(2026-07-19)

- **5/5 mainnet 已关网格三层全对齐**(SKL/1000BONK/0G×2/PTB):建网 byte 精确、成交逐笔、退出归因一致;新代码(v8)格连 pv 触发时刻都对齐(残差 ~2.6bp = 1m 数据物理下限)。
- **待实盘数据补证**:trailing 退出、带持仓再平衡、资金费收费/止损、多线阶梯、破网全链路(建议 testnet 造靶)、exit_reason 分布(≥50 格)——详见对账会话记录。
- 已知残差(全部量化):pv 动作时机(逐 1m vs 900s 节流)、平仓点差、cap 口径(回测恒 1000)、票池源(归档含退市)。
