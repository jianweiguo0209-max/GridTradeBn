# 选币评价分工具 · 使用文档

**版本**:2026-07-22(s030 引擎标定) · **资产目录**:`data/score_research_2026-07-21/`
**性质**:⚠**回顾性评价工具**——给"已经发生的选币"打分,**禁止用作预测排名**
(同一套标签往前看=已判死的打分器,见《选币打分研究总结报告》)。

---

## 1. 它回答什么问题

给任意一组选币(某因子组合的 top-K、或实盘某周期的实际选中币)出一张**体检单**:
这组币的"网格适配度"相对全池是好是坏、好坏多少 bp、坏在哪个轴。
它把选币质量与 pnl 运气解耦:pnl 有噪声,路径标签没有。

## 2. 两个分数 + 一条汇率

### calib_score(主分,量纲=预期引擎 bp)
```
calib_score = 32.82 · z(cross1) − 60.67 · z(drift)
```
- `cross1` 燃料 = 未来12h 收盘价穿越 1% 对数阶梯次数;`drift` 毒药 = |净漂移|
- z = **逐 rt 截面标准化**(同池同期比,regime 自动中和)
- 权重来源:s030 引擎 2,796 格标定回归(bp/σ),四窗符号稳定,R²=0.21
- 权重文件:`ablation/score_eval_weights.json`
- 评价一组选币:`top 的 calib 均值 − 全池 calib 均值` = **边际(bp/格)**

### 燃料否决门(必配,防静币退化角)
```
rank(cross1) < 全池 P25 → 该格判"死水",否决
```
线性 calib 分会被超静币骗(毒药项 −60.7×深负 z_drift 抬分)——**边际与否决率必须并读**。
实证:E单组合边际全场最好看(−2.1)但否决率 64%,选的全是白挂死水币。

### 汇率(一个数读懂整张单)
```
汇率 = top 选中桶的 z(drift)均值 / z(cross1)均值
盈亏平衡线 = 32.82/60.67 = 0.54
```
选币本质是"用漂移买活跃度";引擎牌价说 1 份燃料最多值 0.54 份毒药。
**实付汇率 < 0.54 = 买得赚;> 0.54 = 买贵了**,超线幅度≈边际负多少。

## 3. 体检单读法(红绿灯)

| 指标 | 🟢 | 🟡 | 🔴 |
|---|---|---|---|
| 汇率 | <0.54 | 0.54~0.80 | >0.80 |
| calib 边际 | ≥0 | 0~−2bp | <−2bp |
| 燃料否决率 | <10% | 10~20% | >20% |

现役 rank_sum 基线(2026-07 标定,四窗):汇率 0.62~0.96、边际 −0.7~−2.5bp、否决率 12~16%
——机会成本量级 ≈ −1.5bp/格 ≈ $80/月@当前规模。**巡检只需盯汇率**:爬向 0.8+ = 选币退化
报警;新因子组稳定压到 0.54 以下 = 才真正值得考虑换。

## 4. 常用操作

### 4.1 评价一个因子组合(标签门,秒级)
参照 `combo_scan.py`:定义 `(因子, 方向, 权重)` 列表加入 `COMBOS`,运行即得四窗体检单。
```
.venv/bin/python data/score_research_2026-07-21/combo_scan.py
```

### 4.2 引擎门验证(分钟~小时级)
标签门的赢家**必须**过引擎门(标签赢≠引擎赢,Phase2 血训):参照 `combo_engine_check.py`
——把候选组合的真实 top-K 抽样喂给 s030 引擎,比引擎 pnl 排序与标签边际排序的一致性。

### 4.3 评价实盘某周期的实际选中币
换仓 12h 后,对选中 (rt, symbol) 计算实现标签(标签构造见 `holdout_gate.py` 的
`_label_one`,1% 对数阶梯,日历分钟对齐),按 §2 公式对全池算边际/汇率/否决率。
提前退出的格,严谨口径应把标签截断到退出时刻(否则量的是仓位没经历的路径)。

### 4.4 新窗口重建标签/因子面板
`holdout_gate.py` 的 stage L/F 可对任意日期窗重建(全票池×小时级,PIT;
~10 分钟/双月窗@2workers)。市场基准指数:`scripts/build_market_r15.py`。

## 5. 权重再标定政策

权重绑定引擎配置(退出结构变→路径→pnl 映射变)。**引擎参数再升级时必须重标定**:
参照 `s030_calib.py`(同 2,796 格,几何/止损取 config 现值,pv_pnl_thr 显式传入),
再用 `score_eval.py` 的 A 段拟合。⚠只用 cross1+drift 两个原始轴回归——
加 eff/mae 会因构造性共线导致 β 逐窗换号(eff=cross1/(1+100·mae),drift≤mae)。

## 6. 边界与禁忌(违者结论无效)

1. **禁作预测**:分数用的是实现标签(事后),前视语义=已判死的打分器。
2. **群体判断,不判单格**:R²=0.21,单格好形状仍可能亏(时序×止损交互);
   ≥几十格聚合才可靠。
3. **评价路径,不评价决策**:好决策也会抽到坏路径;评选币**策略**须多期聚合洗运气。
4. **标签边际 ≠ 组合收益**:无 offset 结构/资金分配/同币锁;组合级结论要走真回测。
5. **三道门纪律**:标签门(本工具)→引擎门→组合战役+双留出。任何"发现更优组合"
   的宣称,止步于哪道门就只有哪道门的效力。
