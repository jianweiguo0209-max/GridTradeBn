# 选币逻辑：未来波动率预测因子设计文档

> 目标：在网格选币中增加一个**预测因子**，预测某个币在未来窗口期的波动性，从而优先选出更适合网格交易的标的。

## 1. 背景与可行性

### 1.1 为什么和网格策略契合
网格策略的本质是"价格在区间内来回震荡越多越赚钱"。因此真正想选的是：

> **未来窗口期内实现波动率高、但方向性趋势（单边漂移）小的币。**

预测未来波动率正好命中网格选币的核心需求。

### 1.2 为什么可行
波动率是金融时间序列里最容易预测的量之一（**波动率聚集 / volatility clustering**，自相关很强）：
- 预测涨跌**方向**很难；
- 预测**振幅大小**相对容易——当前波动率会持续到下一期。

因此**不一定需要机器学习**，简单统计模型就能产生有效的预测因子。

### 1.3 与现有因子的关系
当前代码已有**向后看（实际值）**的波动率类因子：
- `Atr_signal` —— ATR 去量纲（`account_0/utils/fancy_grid_function.py:450`）
- `db_volume_v1_signal` —— 成交量活跃度（`account_0/utils/fancy_grid_function.py:7`）

本设计新增的是**向前看（预测值）**的因子，二者互补。

### 1.4 关于 OKX API
- 永续合约**没有**直接可取的隐含波动率（IV 只有期权品种才有）。
- 因此未来波动率需用**已获取的 OHLCV 数据**（`max_candle_num=160` 根、`period=12H`，见 `account_0/config.py:35`）做**统计预测**。
- 优势：能直接从 `df` 计算，几乎零额外请求成本。
- 如需更高精度，可额外用 OKX `GET /api/v5/market/candles` 拉取细粒度 K 线（1H/15m）来稳定估计。

## 2. 实现方案（分阶段）

### 阶段一：区间估计量 + EWMA 持续性预测（推荐，立即可做）

用 high/low 的 **Parkinson 区间估计量**（比 close-to-close 效率高数倍），再用 **EWMA** 平滑得到"下一期波动率的一步预测"，并惩罚单边趋势以突出"震荡"而非"方向"。

完全契合现有 `*_signal(df, n, diff_num, factor_name)` 模式：

```python
def VolForecast_signal(*args):
    """
    未来窗口实现波动率预测因子（网格场景：越高越好）
    用 Parkinson 区间估计量 + EWMA 持续性做一步预测，
    并惩罚单边趋势，突出"震荡"而非"方向"。
    """
    df = args[0]
    n = args[1]
    diff_num = args[2]
    factor_name = args[3]

    # Parkinson 估计量：high/low 对数区间（比 close-to-close 更高效）
    log_hl = np.log(df['high'] / (df['low'] + eps))
    park_var = (log_hl ** 2) / (4 * np.log(2))
    realized_vol = np.sqrt(park_var.rolling(n, min_periods=1).mean())

    # EWMA 利用"波动率持续性"预测下一期（RiskMetrics 思路）
    vol_forecast = realized_vol.ewm(span=n, adjust=False).mean()

    # 惩罚趋势：振幅大但不单边的币更好（震荡质量）
    drift = (df['close'] / df['close'].shift(n) - 1).abs()
    osc_quality = vol_forecast / (drift + eps)

    df[factor_name] = osc_quality
    return df
```

### 阶段二：GARCH(1,1)（提升精度）

用 `arch` 库做正经的条件方差模型，是一步预测波动率的经典做法。
- 优点：理论扎实，对波动率聚集建模更精确。
- 代价：每个币单独拟合、计算较重；160 根 K 线拟合不稳，需额外取细粒度数据（1H/15m）稳定估计。

### 阶段三：机器学习回归（效果上限最高）

特征（历史波动率、成交量、区间、时段等）→ 回归下一窗口的实现波动率。
- 优点：可融合多源信息，效果上限最高。
- 代价：需离线训练、防止 lookahead bias、定期重训的工程支撑。
- **建议：先用阶段一验证有效后再上。**

## 3. 接入步骤（阶段一）

1. **新增因子函数**：在 `account_0/utils/fancy_grid_function.py` 增加 `VolForecast_signal`。
2. **计算因子**：在 `cal_factor`（`account_0/utils/fancy_grid_function.py:511`）中调用，例如：
   ```python
   VolForecast_signal(df, 5, 0, 'VolForecast_5')
   ```
3. **加入选币因子字典**：更新 `account_0/startup.py:50` 和 `account_0/config.py:39` 的 `factors` / `weight_list`。

### ⚠️ 排序方向（关键）
在 `select_grid_coin`（`account_0/utils/fancy_grid_function.py:596-604`）里：
- `factor_info[factor]` 会传给 `rank(ascending=...)`；
- 最终 `rank_sum` **越小越会被选中**。

因此要"选预测波动率高的币"，**必须把该因子设成 `ascending=False`**（高值 → rank=1 → 小 rank_sum → 被选）：

```python
# 示例
strategy_config['factors'] = {"Reg_v2_2": True, "Sgcz_2": True, "VolForecast_5": False}
strategy_config['weight_list'] = [1, 1, 1]   # 元素个数必须与因子数一致
```

> 注意：`weight_list` 的元素个数必须与因子个数对应（`account_0/config.py:39`）。

## 4. 关键前提与风控

- **严禁 lookahead bias**：因子只能用当前 K 线及之前的信息（沿用现有函数的 `rolling` / `shift` 习惯即可）。
- **必须回测验证**：加因子后一定回测确认"网格的实际 PnL 是否改善"。**预测准 ≠ 选币有效。**
- **AB 测试上线**：因为是实盘代码，建议用单独 offset 做 AB 测试（类似 `grid_version` 的 AB 框架，见 `account_0/config.py:47`），确认无误后再全量。
